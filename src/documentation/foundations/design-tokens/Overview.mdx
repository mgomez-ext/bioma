# Design Tokens

Design tokens are the single source of truth for design decisions in Bioma. They're named values that represent visual design attributes like colors, spacing, typography, and more.

## What are Design Tokens?

Tokens are named entities that store visual design information. Instead of hardcoding values like `#E30038` or `16px`, you use tokens like `--primary-default` or `--spacing-4`.

### Without Tokens
```scss
.button {
  background: #E30038;      // ‚ùå What is this color?
  padding: 12px 24px;       // ‚ùå Why these values?
  border-radius: 8px;       // ‚ùå Is this consistent?
}
```

### With Tokens
```scss
.button {
  background: var(--primary-default);  // ‚úÖ Clear purpose
  padding: var(--spacing-3) var(--spacing-6);  // ‚úÖ Consistent scale
  border-radius: var(--radius-md);    // ‚úÖ System value
}
```

## Token Architecture

Bioma uses a **two-level token system** inspired by design systems like Material Design and Polaris:

```
Foundation Tokens (Level 1)
         ‚Üì
Semantic Tokens (Level 2)
         ‚Üì
Components
         ‚Üì
UI
```

### Level 1: Foundation Tokens

**Purpose**: Define primitive, context-independent values

**Characteristics**:
- Absolute values
- Never reference other tokens
- Platform agnostic
- Rarely change

**Examples**:
```scss
--brand-500: #E30038;
--neutral-950: #121212;
--spacing-4: 16px;
--radius-md: 8px;
```

**Naming Pattern**: `{category}/{value}` or `{category}-{value}`

### Level 2: Semantic Tokens

**Purpose**: Provide contextual meaning by referencing foundation tokens

**Characteristics**:
- Reference foundation tokens
- Change based on context (light/dark mode)
- Self-documenting
- Frequently used in components

**Examples**:
```scss
--primary-default: var(--brand-500);
--text-primary: var(--neutral-950);
--bg-default: var(--neutral-white);
```

**Naming Pattern**: `{purpose}-{variant}`

## Dark Mode with Tokens

Semantic tokens automatically adapt to light/dark themes:

```scss
// Light Mode (default)
:root {
  --text-primary: var(--neutral-950);  // Dark text
  --bg-default: var(--neutral-white);   // Light background
  --border-default: var(--neutral-300); // Light border
}

// Dark Mode (override)
[data-theme="dark"] {
  --text-primary: var(--neutral-100);   // Light text
  --bg-default: var(--neutral-950);     // Dark background
  --border-default: var(--neutral-700); // Dark border
}
```

Components using semantic tokens work in both modes **without code changes**!

## Token Usage

### In CSS/SCSS
```scss
.card {
  background: var(--bg-default);
  border: 1px solid var(--border-default);
  border-radius: var(--radius-lg);
  padding: var(--spacing-6);
  box-shadow: var(--shadow-md);

  &__title {
    color: var(--text-strong);
    font-size: var(--font-2xl);
    font-weight: var(--font-semibold);
    margin-bottom: var(--spacing-3);
  }

  &__content {
    color: var(--text-secondary);
    line-height: var(--leading-relaxed);
  }
}
```

### In Figma
Use token names in:
- Layer names: `text/primary`, `spacing-4`
- Text styles: Reference token values
- Color styles: Map to token names
- Variables: Connect to token system

## Best Practices

### ‚úÖ Do

**Use semantic tokens in components**:
```scss
color: var(--text-primary);  // ‚úÖ
```

**Provide fallback values**:
```scss
color: var(--text-primary, #121212);  // ‚úÖ
```

**Use tokens for all visual properties**:
```scss
.component {
  color: var(--text-primary);
  padding: var(--spacing-4);
  margin-bottom: var(--spacing-6);
  border-radius: var(--radius-md);
  transition: all var(--transition-fast);
}
```

### ‚ùå Don't

**Don't hardcode values**:
```scss
color: #121212;  // ‚ùå
padding: 16px;   // ‚ùå
```

**Don't use foundation tokens in components**:
```scss
color: var(--neutral-950);  // ‚ùå Use --text-primary
background: var(--brand-500);  // ‚ùå Use --primary-default
```

**Don't create one-off tokens**:
```scss
--my-special-blue: #0066CC;  // ‚ùå Use existing tokens
```
