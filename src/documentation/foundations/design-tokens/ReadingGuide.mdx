# Reading Design Tokens

Learn how to read and understand Bioma's token names to use them effectively in your work.

## Token Anatomy

Every token name follows a consistent structure that tells you what it is and how to use it.

```
┌─────────────┬──────────┐
│  Category   │  Scale   │  Foundation Token
└─────────────┴──────────┘
   --brand   -   500

┌─────────────┬──────────┐
│  Purpose    │ Variant  │  Semantic Token
└─────────────┴──────────┘
   --text   -  primary
```

---

## Foundation Token Names

Foundation tokens use **category + scale** naming:

### Pattern: `--{category}-{scale}`

**Categories** identify the token type:
- `brand` - Brand color palette
- `neutral` - Grayscale palette
- `spacing` - Spacing values
- `radius` - Border radius
- `font` - Typography values
- `shadow` - Elevation shadows

**Scales** indicate position in the range:

**Color Scales** (50-950):
```scss
--brand-50    // Lightest (near white)
--brand-500   // Middle (pure hue)
--brand-950   // Darkest (near black)
```
- Lower numbers = lighter
- Higher numbers = darker
- `500` is typically the "truest" color

**Spacing Scales** (0-16):
```scss
--spacing-0   // 0px
--spacing-4   // 16px (base × 4)
--spacing-16  // 64px (base × 16)
```
- Numbers represent multiples of 4px base unit
- Not all multiples exist (gaps like 7, 11, 13-15)

**Size Scales** (xs-5xl):
```scss
--font-xs     // Smallest
--font-base   // Standard (16px)
--font-5xl    // Largest
```
- `xs`, `sm`, `md`, `lg`, `xl`, `2xl`, `3xl`, `4xl`, `5xl`
- `base` replaces `md` for font sizes

---

## Semantic Token Names

Semantic tokens use **purpose + variant** naming:

### Pattern: `--{purpose}-{variant}`

**Purposes** describe the UI element:

**Text Purposes**:
```scss
--text-{variant}     // Text colors
```

**Background Purposes**:
```scss
--bg-{variant}       // Background colors
```

**Border Purposes**:
```scss
--border-{variant}   // Border colors
```

**Action Purposes**:
```scss
--primary-{variant}      // Brand actions
--secondary-{variant}    // Secondary actions
--tertiary-{variant}     // Tertiary actions
```

**Status Purposes**:
```scss
--success-{variant}  // Success states
--warning-{variant}  // Warning states
--error-{variant}    // Error states
--info-{variant}     // Informational states
```

**Variants** describe the intensity or state:

**Hierarchy Variants** (text, backgrounds):
```scss
--text-strong        // Highest emphasis
--text-primary       // Default emphasis
--text-secondary     // Lower emphasis
--text-tertiary      // Subtle emphasis
--text-muted         // Very subtle
--text-disabled      // Inactive state
```

**Interactive Variants** (actions, statuses):
```scss
--primary-default    // Resting state
--primary-hover      // Mouse hover
--primary-active     // Click/press
--primary-disabled   // Inactive state
```

**Appearance Variants**:
```scss
--primary-subtle     // Light/background variant
--primary-strong     // High contrast/text variant
```

**Special Variants**:
```scss
--text-inverse       // For dark backgrounds
--border-inverse     // For dark backgrounds
```

## Reading Examples

Let's decode some token names:

### Example 1: `--brand-500`

```
--brand-500
  │     │
  │     └─ Scale: Middle of the range
  └─────── Category: Brand color palette
```

**What it is**: The primary Bioma red color (`#E30038`)
**When to use**: Never directly! Use `--primary-default` instead
**Type**: Foundation token (building block)

---

### Example 2: `--text-primary`

```
--text-primary
  │     │
  │     └─ Variant: Primary emphasis
  └─────── Purpose: Text color
```

**What it is**: Default body text color
**When to use**: Paragraphs, primary content, labels
**Type**: Semantic token (use in components)
**Value**: References `--neutral-950` in light mode, `--neutral-50` in dark mode

---

### Example 3: `--primary-hover`

```
--primary-hover
  │        │
  │        └─ Variant: Hover state
  └────────── Purpose: Primary brand action
```

**What it is**: Background color for brand buttons on hover
**When to use**: `:hover` state on primary buttons and actions
**Type**: Semantic token
**Value**: References `--brand-600` (darker than default)

---

### Example 4: `--spacing-4`

```
--spacing-4
  │       │
  │       └─ Scale: 4 units (16px)
  └───────── Category: Spacing
```

**What it is**: Standard spacing value (16px = 4 × 4px base)
**When to use**: Padding, margins, gaps
**Type**: Foundation token (but okay to use directly)
**Note**: Spacing tokens are pure values, safe to use anywhere

---

### Example 5: `--success-subtle`

```
--success-subtle
  │        │
  │        └─ Variant: Subtle/light version
  └────────── Purpose: Success status
```

**What it is**: Light green background for success messages
**When to use**: Success alert backgrounds, success badges
**Type**: Semantic token
**Value**: References `#D1FAE5` (light green)

## Common Patterns

### Color Hierarchy

Text colors follow a hierarchy from strongest to weakest:

```scss
--text-strong      // Headings (darkest)
--text-primary     // Body text
--text-secondary   // Supporting text
--text-tertiary    // Labels, captions
--text-muted       // Placeholders (lightest)
```

**Tip**: Match text emphasis to content importance

---

### Interactive State Progression

Interactive elements follow a progression:

```scss
--primary-default   // Resting (base state)
--primary-hover     // Mouse over (slightly darker)
--primary-active    // Clicking (darkest)
--primary-disabled  // Inactive (grayed out)
```

**Tip**: Each state should be visually distinct but related

---

### Appearance Variations

Many tokens have `subtle` and `strong` variants:

```scss
--primary-subtle    // Light background (tinted)
--primary-default   // Standard appearance
--primary-strong    // High contrast (white text)
```

**Subtle**: Light backgrounds, ghost variants
**Strong**: Text on colored backgrounds, high contrast

## Token Selection Guide

### "Which token should I use?"

**For body text:**
```scss
color: var(--text-primary);  ✅
```

**For headings:**
```scss
color: var(--text-strong);  ✅
```

**For helper text:**
```scss
color: var(--text-secondary);  ✅
```

**For disabled text:**
```scss
color: var(--text-disabled);  ✅
```

---

**For white backgrounds:**
```scss
background: var(--bg-default);  ✅
```

**For subtle containers:**
```scss
background: var(--bg-subtle);  ✅
```

**For alternating rows:**
```scss
background: var(--bg-muted);  ✅
```

---

**For standard borders:**
```scss
border-color: var(--border-default);  ✅
```

**For very subtle dividers:**
```scss
border-color: var(--border-subtle);  ✅
```

**For emphasized borders:**
```scss
border-color: var(--border-strong);  ✅
```

---

**For brand buttons:**
```scss
background: var(--primary-default);
&:hover { background: var(--primary-hover); }
&:active { background: var(--primary-active); }
```

**For success messages:**
```scss
background: var(--success-subtle);
color: var(--success-strong);
border-color: var(--success-default);
```

## Anti-Patterns

### ❌ Don't use foundation colors in components

```scss
// Wrong
color: var(--neutral-950);
background: var(--brand-500);
```

```scss
// Right
color: var(--text-primary);
background: var(--primary-default);
```

**Why**: Semantic tokens adapt to themes; foundation tokens don't.

---

### ❌ Don't mix hierarchy levels

```scss
// Wrong - using strong text with tertiary emphasis
font-weight: var(--font-bold);
color: var(--text-tertiary);
```

```scss
// Right - matching weight to emphasis
font-weight: var(--font-semibold);
color: var(--text-strong);
```

**Why**: Visual weight should match semantic importance.

---

### ❌ Don't use hover states as defaults

```scss
// Wrong
background: var(--primary-hover);
```

```scss
// Right
background: var(--primary-default);
&:hover { background: var(--primary-hover); }
```

**Why**: Hover states are only for `:hover` pseudo-class.

## Quick Reference

### Token Type Recognition

**It's a foundation token if**:
- Has numeric scale: `-50`, `-500`, `-950`
- Has t-shirt sizing: `-xs`, `-lg`, `-2xl`
- Category is: `brand`, `neutral`, `spacing`, `radius`, `font`, `shadow`

**It's a semantic token if**:
- Has descriptive variant: `-primary`, `-hover`, `-subtle`
- Purpose is: `text`, `bg`, `border`, `primary`, `success`, `error`

### When to Use Which

**Use foundation tokens**:
- Spacing values (`--spacing-4`)
- Border radius (`--radius-md`)
- Typography scales (`--font-lg`)

**Use semantic tokens**:
- All colors (text, backgrounds, borders)
- Interactive states
- Status indicators
- Component styling
