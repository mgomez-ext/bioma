# Fundamentos

Los fundamentos son los bloques de construcción centrales del sistema de diseño Bioma. Definen el lenguaje visual y proporcionan los elementos atómicos sobre los que se construyen todos los componentes y patrones.

## ¿Qué son los Fundamentos?

Los fundamentos incluyen:

- **Design Tokens**: La única fuente de verdad para decisiones de diseño
- **Color**: Colores de marca, neutrales y sistema de color semántico
- **Tipografía**: Familias tipográficas, escalas, pesos y estilos de texto
- **Espaciado**: Escala de espaciado consistente para layouts
- **Layout**: Sistemas de grilla y breakpoints responsivos
- **Border**: Anchos de borde y valores de radius
- **Elevation**: Sistema de sombras para profundidad y jerarquía
- **Animation**: Principios de movimiento y funciones de timing
- **Iconografía**: Sistema de íconos y guías de uso

## Arquitectura de Tokens

Bioma usa un **sistema de tokens de dos niveles**:

### Nivel 1: Foundation Tokens (Primitivos)
Valores crudos que no cambian según el contexto:
```scss
--brand-500: #E30038;
--neutral-950: #121212;
--spacing-4: 16px;
```

### Nivel 2: Semantic Tokens (Contextuales)
Tokens contextuales que referencian foundation tokens:
```scss
--primary-default: var(--brand-500);
--text-primary: var(--neutral-950);
--bg-default: var(--neutral-white);
```

## ¿Por Qué Usar Fundamentos?

### Consistencia
Cuando todos usan los mismos fundamentos, el producto se siente cohesivo y profesional.

### Eficiencia
Diseñadores y desarrolladores no necesitan tomar decisiones de bajo nivel—los fundamentos proporcionan las respuestas.

### Mantenibilidad
Cambiá una vez, actualizá en todos lados. ¿Necesitás ajustar el espaciado? Actualizá el token.

### Escalabilidad
A medida que el sistema crece, los fundamentos aseguran que los nuevos componentes se mantengan consistentes con los existentes.

### Dark Mode
Los tokens semánticos se adaptan automáticamente a temas claros y oscuros.

## Usando Fundamentos

### En Diseño (Figma)
```
Usá nombres de tokens como nombres de capas o en estilos de texto:
- text/primary
- spacing-4
- brand/500
```

### En Código (CSS/SCSS)
```scss
.component {
  color: var(--text-primary);
  padding: var(--spacing-4);
  background: var(--bg-default);
  border-radius: var(--radius-md);
}
```

### En Componentes (Angular)
```typescript
// Los componentes heredan automáticamente los tokens
// Solo usá los nombres de tokens semánticos
```

## Convención de Nombres de Tokens

Los tokens de Bioma siguen un patrón de nombres claro:

### Foundation Tokens
```
{category}/{value}
```
Ejemplos:
- `brand/500` - Rojo de marca en peso 500
- `neutral/950` - Neutral más oscuro
- `spacing-4` - Espaciado de 16px

### Semantic Tokens
```
{purpose}-{variant}
```
Ejemplos:
- `text-primary` - Color de texto primario
- `bg-default` - Fondo predeterminado
- `primary-hover` - Estado hover de acción primaria

## Estrategia de Dark Mode

```scss
// Light Mode (predeterminado)
:root {
  --text-primary: var(--neutral-950); // Texto oscuro
  --bg-default: var(--neutral-white);  // Fondo claro
}

// Dark Mode (override)
[data-theme="dark"] {
  --text-primary: var(--neutral-100);  // Texto claro
  --bg-default: var(--neutral-950);    // Fondo oscuro
}
```

¡Los componentes se adaptan automáticamente sin cambios en el código!

## Mejores Prácticas

### ✅ Sí
- Usá siempre tokens semánticos en componentes
- Referenciá foundation tokens en tokens semánticos
- Usá nombres de tokens consistentemente entre diseño y código
- Documentá por qué existen los tokens

### ❌ No
- Hardcodear valores en componentes
- Saltear tokens semánticos y usar foundation tokens directamente en componentes
- Crear tokens para usos únicos
- Usar nombres genéricos como "color1" o "spacing-A"
