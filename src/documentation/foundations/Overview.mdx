# Foundations

Foundations are the core building blocks of the Bioma design system. They define the visual language and provide the atomic elements that all components and patterns are built upon.

## What are Foundations?

Foundations include:

- **Design Tokens**: The single source of truth for design decisions
- **Color**: Brand colors, neutrals, and semantic color system
- **Typography**: Font families, scales, weights, and text styles
- **Spacing**: Consistent spacing scale for layouts
- **Layout**: Grid systems and responsive breakpoints
- **Border**: Border widths and radius values
- **Elevation**: Shadow system for depth and hierarchy
- **Animation**: Motion principles and timing functions
- **Iconography**: Icon system and usage guidelines

## Token Architecture

Bioma uses a **two-level token system**:

### Level 1: Foundation Tokens (Primitives)
Raw values that don't change based on context:
```scss
--brand-500: #E30038;
--neutral-950: #121212;
--spacing-4: 16px;
```

### Level 2: Semantic Tokens (Contextual)
Contextual tokens that reference foundation tokens:
```scss
--primary-default: var(--brand-500);
--text-primary: var(--neutral-950);
--bg-default: var(--neutral-white);
```

## Why Use Foundations?

### Consistency
When everyone uses the same foundations, the product feels cohesive and professional.

### Efficiency
Designers and developers don't need to make low-level decisions—foundations provide the answers.

### Maintainability
Change once, update everywhere. Need to adjust spacing? Update the token.

### Scalability
As the system grows, foundations ensure new components stay consistent with existing ones.

### Dark Mode
Semantic tokens automatically adapt to light and dark themes.

## Using Foundations

### In Design (Figma)
```
Use token names as layer names or in text styles:
- text/primary
- spacing-4
- brand/500
```

### In Code (CSS/SCSS)
```scss
.component {
  color: var(--text-primary);
  padding: var(--spacing-4);
  background: var(--bg-default);
  border-radius: var(--radius-md);
}
```

### In Components (Angular)
```typescript
// Components automatically inherit tokens
// Just use the semantic token names
```

## Token Naming Convention

Bioma tokens follow a clear naming pattern:

### Foundation Tokens
```
{category}/{value}
```
Examples:
- `brand/500` - Brand red at 500 weight
- `neutral/950` - Darkest neutral
- `spacing-4` - 16px spacing

### Semantic Tokens
```
{purpose}-{variant}
```
Examples:
- `text-primary` - Primary text color
- `bg-default` - Default background
- `primary-hover` - Primary action hover state

## Dark Mode Strategy

```scss
// Light Mode (default)
:root {
  --text-primary: var(--neutral-950); // Dark text
  --bg-default: var(--neutral-white);  // Light bg
}

// Dark Mode (override)
[data-theme="dark"] {
  --text-primary: var(--neutral-100);  // Light text
  --bg-default: var(--neutral-950);    // Dark bg
}
```

Components automatically adapt without code changes!

## Best Practices

### ✅ Do
- Always use semantic tokens in components
- Reference foundation tokens in semantic tokens
- Use token names consistently across design and code
- Document why tokens exist

### ❌ Don't
- Hardcode values in components
- Skip semantic tokens and use foundation tokens directly in components
- Create tokens for one-off uses
- Use generic names like "color1" or "spacing-A"
